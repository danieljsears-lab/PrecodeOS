## Purpose

This file is retained as a compatibility pointer. It no longer defines a separate student workflow.

Use `tasks/templates/PRODUCT-IDEATION-WORKBOOK.md` instead. The default Product Ideation Workbook now owns the beginner idea-to-Conviction Packet path and includes the useful concepts from this former experimental workbook:

- builder lenses for operators/PMs, solo B2B founders, and AI-curious creators
- smallest complete useful payoff
- visible iteration
- core workflow spine
- student-friendly self-checks
- evidence-only Conviction Packet handoff into Local Source Intake

## Boundary

The Product Ideation Workbook remains evidence only. It is not a PRD, not product approval, not task activation, not an implementation plan, not a bead, and not permission to code.

Bring the reviewed Conviction Packet into Precode Local Source Intake before asking Precode to update `PRODUCT.md`, draft a PRD, propose beads, or implement anything.

## Replacement Prompt

Use this instead of the old Student Idea To MVE prompt:

```text
Use the Product Ideation Workbook as my guided PrecodeOS product coach.

Help me choose the closest builder lens, clarify one rough idea, produce a Product Brief after at most three high-level product or business questions, challenge weak assumptions, rate evidence strength, name the weakest assumption, identify the smallest complete useful payoff, and produce a Conviction Packet for Local Source Intake.

Treat the workbook as evidence only. Do not write code, create a PRD, create beads, update PRODUCT.md, create a roadmap or backlog, or treat the packet as implementation approval.
```
