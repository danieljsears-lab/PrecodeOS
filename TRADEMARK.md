## Name Meaning Vs. Trademark Use

“Precode” names the work that must happen before code, and “OS” describes the small operating layer around the agent and repo. That meaning explains the project name, but it does not grant trademark rights or permission to present a fork, derivative, product, service, or organization as official PrecodeOS.

## Allowed Use

Truthful, non-confusing references are allowed, including:

- “compatible with PrecodeOS”
- “forked from PrecodeOS”
- “uses PrecodeOS”
- “based on PrecodeOS”

These uses must not imply official endorsement, ownership, project continuity, or maintainer approval.

## Forks And Derivatives

Forks and derivatives should use clearly distinct names unless Dan Sears / Recode gives written permission to use the PrecodeOS name as the primary project or product name.

Do not use “PrecodeOS” or “Precode” in a way that could confuse users about which project is official, who maintains it, or where the canonical project lives.
